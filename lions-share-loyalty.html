How the Program Works

Lion's Share Loyalty Program is a monthly rebate program tied to our membership subscription. Members can earn back up to $19.99, their full membership fee, by completing two simple tasks each month:

Making purchases with RoarMoney
Logging into the app daily
Bronze $2
Silver $5
Gold $10
Platinum $19.99

The Problem

Despite the program existing for years, there was no dedicated experience for it at all. Cashback was deposited straight into the users' Rewards History, with no explanation of why they received that amount or what they'd need to do differently next month.

An incentive program nobody could see

The reward arrived with zero context — no tier name, no breakdown of what was completed, no sense of whether $5 was a good or bad result. Sitting alongside unrelated cashback transactions, it read as just another small deposit rather than the outcome of a program designed to motivate behaviour.

Outcome

+7.6% Increase in average RoarMoney purchases
+0.7% Increase in average login days per month

Decision 1: From Buried to Visible

From a buried line item to a live progress widget

With no existing UI to build on, the visual direction drew from how games communicate leveling up, clear tiers that read as milestones, visual feedback when progress is made, and a sense of forward momentum rather than a static checklist.

Before vs After
Progress widget

The widget needed to make three things obvious at a glance: which tier the user is currently in, what they've already done, and what's left to do before the month ends.

Before: buried in Rewards history
After: dedicated progress widget
Improvement

Progress is shown as a tier path with checkpoints already passed, the users see both how far they've come and what's directly ahead, not just an abstract dollar amount. The gift box motif reframes cashback as something earned and unwrapped, not just deposited.

1. Progress at a glance (what's done, what's next, and how much they'll earn)

Users can see exactly where they stand in real time: tasks completed, current tier, and the cashback amount they're on track to receive by end of month. Tapping into the next tier is always visible, removing any guesswork about what they need to do to earn more.

2. A moment of delight when a tier is achieved

When users reach a new tier, the gift box animates, unwrapping to celebrate the achievement. A small but intentional moment that makes progress feel rewarding rather than transactional, consistent with the gamified direction of the whole widget.

3. Refresh date keeps urgency real

The countdown to the next cycle reset (Refreshes in X days) tells users exactly how much time they have left to complete their tasks. It creates natural urgency without being pushy, users who are close to the next tier know when they need to act.

Decision 2: Edge Cases

The widget had to communicate more than progress

Beyond the four tiers, the widget needed to handle edge cases — like a suspended Credit Builder Plus membership, where Lion's Share benefits pause too.

Edge cases
Progress widget

Designing for the happy path is straightforward. The harder work was making sure every other state, achieving a tier, reaching Platinum, or having a suspended membership, communicated the right thing clearly and didn't leave users confused about where they stood.

Active state
Achieving state
Inactive state
Improvement

The widget needed to work correctly across every possible membership state, not just the happy path of active progress. Each state communicates a different message to the user, from encouragement to pause to celebration.

1. Active states reflect real progress, not just position

A user at Silver with 5/10 purchases sees something different from a user at Silver with 10/10, the state is specific to where they actually are, not just which tier they've reached.

2. Suspended state protects user trust

Rather than silently showing zero progress, the suspended state tells users explicitly why the widget is inactive, preventing confusion about whether their progress has been lost.

3. Achieving state creates a motivating moment

The closest thing to a push notification without being one, surfacing when a user is nearly at the next tier, right when a last-minute action is most likely.

Implementation

Build with Rive State Machine

No precedent to follow

This was the first State Machine implementation at MoneyLion. There was no internal playbook, the approach to structuring states and engineer-controlled inputs was worked out collaboratively, in real time, with the frontend engineer.

Engineer-controlled inputs kept the system flexible

Rather than triggering animations from hardcoded states, the state machine exposes inputs engineers can drive directly from live data (tier, progress percentage, and suspended status) all feed in without needing new animation assets per scenario.

Iteration required real back-and-forth

Without a reference point, building this involved genuine trial and error with engineering, testing what was technically feasible, what needed more research, and how to structure states so they'd integrate cleanly into the existing codebase.

What's Next

Even with the refinements shipped, there's room to go further. These are the areas actively being explored.

Increase awareness of the program through a stronger in-app and marketing push

The engagement lift from the widget proves the program works when users can see it. The next step is making sure more users find it in the first place, through better in-app placement and targeted outreach to members who are enrolled but haven't yet participated.
